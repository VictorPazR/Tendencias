Storage

  • Unstaffed data center

    Es el espacio donde se concentran los recursos necesarios para el procesamiento de la información de una organización. También se conoce como centro de cómputo en Hispanoamérica y en España como centro de cálculo, centro de datos, centro de proceso de datos o centro de informática.

  • Phase Change Memory (PCM)

    Es un tipo de memoria no volátil, lo que significa que cuando apagamos el PC o hay un corte de luz, no pierde los datos que contiene, con las ventajas que esto representaría para un PC, como reinicios ultra rápidos o volver a abrir todo exactamente como lo teníamos al apagar y encender el PC.

  • Flash storage and All-Flash Arrays

    Una matriz basada íntegramente en tecnología flash (AFA) es una infraestructura de almacenamiento que contiene solo discos de memoria flash en lugar de discos duros giratorios. El almacenamiento basado íntegramente en tecnología flash también suele llamarse matriz de estado sólido (Solid-State Array, SSA). Las AFA y los SSA ofrecen velocidad, rendimiento y agilidad para sus aplicaciones empresariales.

  • Persistent Memory

    Es un dispositivo de memoria direccionable según los bytes, de alto rendimiento y estado sólido que reside en el bus de memoria. Al estar en el bus de memoria, la PMEM accede a los datos de forma similar a la DRAM, lo que significa que tiene prácticamente la misma velocidad y latencia de la DRAM y la no volatilidad de la memoria flash NAND. El NVDIMM (módulo de memoria inline doble no volátil) y los DIMM Intel 3D XPoint (también conocidos como módulos de memoria persistente Optane DC) son dos ejemplos de tecnología de memoria persistente.

  • Hybrid multi-cloud storage

    El almacenamiento en la nube híbrida es cuando algunos datos empresariales se almacenan en nubes públicas, mientras que otros datos permanecen dentro de nubes privadas o bases de datos locales. Las dos infraestructuras distintas están unidas dentro del panorama de datos de una organización por tecnología que permite la portabilidad de datos.

  • End-to-end NVMe

    NVMe es un protocolo de almacenamiento diseñado específicamente para almacenamiento basado en flash. Está basado en PCIe y proporciona más comandos por cola de E/S y más colas de E/S. La extensión NVMe over Fabrics (NVMe-F) brinda a las redes IP y Fibre Channel (FC) la capacidad de aprovechar el mayor número de comandos y la profundidad de cola de NVMe para aprovechar al máximo el almacenamiento basado en memoria. Como se discutió en nuestro blog '¿Qué es Scale-Out NVMe?', NVMe-F se usará primero para crear una arquitectura de almacenamiento más escalable, pero eventualmente, se abrirá paso en servidores y conmutadores, construyendo un punto a punto. Arquitectura NVMe.

  • Storage Class Memory (SCM)

    Es un tipo de memoria no volátil que incluye una fuente de energía que evita la pérdida de los datos si hay un error en el sistema o un corte en el suministro eléctrico. Por lo demás, se comporta como una memoria DRAM

  • Software Defined Storage

    Es un término de marketing para el software de almacenamiento de datos informáticos para el aprovisionamiento basado en políticas y la gestión del almacenamiento de datos independientemente del hardware subyacente.

  • Predictive Storage

    Puede jugar un papel importante en llevar su pila de almacenamiento al siguiente nivel. Los algoritmos de aprendizaje automático pueden atravesar los datos y descubrir información crítica. Al analizar los patrones de datos, puede mejorar la planificación de la capacidad de su infraestructura de almacenamiento y acelerar su eficiencia y rendimiento.

  • Fully Automated Storage Tiering (AST)

    Es una función de administración de software de almacenamiento que mueve información dinámicamente entre diferentes tipos de disco y niveles de RAID para cumplir con los requisitos de espacio, rendimiento y costo.

  • Data lake

    Es un repositorio de almacenamiento que contienen una gran cantidad de datos en bruto y que se mantienen allí hasta que sea necesario. A diferencia de un data warehouse jerárquico que almacena datos en ficheros o carpetas, un data lake utiliza una arquitectura plana para almacenar los datos.

  • Hyper-converged storage

    Es un sistema unificado y definido por software que reúne todos los elementos de un centro de datos tradicional: almacenamiento, recursos informáticos, red y gestión.

  • In memory computing

    Se trata de una manera de almacenar la información diferente a la convencional. Como sabrás, lo normal es enviar los datos a un disco duro para su almacenamiento. Sin embargo, a la hora de recuperar lo guardado, el proceso es lento en comparación con otras alternativas.

  • Automated Data Classification

    Es donde su software de clasificación de datos solicita o apoya a los usuarios en la clasificación correcta y precisa de sus archivos de datos.
    Hay varias formas en que se puede utilizar la clasificación automatizada para garantizar un éxito óptimo en la clasificación de datos. La clasificación automatizada puede ser;
    • Completamente automatizado
    • Sugirió
    • Prescrito
    • Avalado por el usuario
    Se basa en información como dónde se almacenan los datos, cómo se comparten, cuándo se crearon e incluso el análisis de contenido para clasificar los datos correctamente.

  • DNA data storage

    Es el proceso de utilizar moléculas de ADN como medio de almacenamiento. A diferencia de las formas ópticas y magnéticas de las tecnologías de almacenamiento presentes en la actualidad, los datos de ADN no se almacenarán en dígitos binarios (es decir, 1 y 0). En cambio, se codificarían en bases de nucleótidos de ADN (A, C, G, T) y se almacenarían. Estos hilos luego se convierten en dígitos binarios cuando es necesario. En este momento, existen más de 11 billones de gigabytes de datos, y cada día se agregan al menos 2,5 millones de gigabytes más. Los medios de almacenamiento de datos disponibles en el mundo no pueden seguir el ritmo de este aumento masivo. El almacenamiento de ADN es una solución a este problema de almacenamiento.
